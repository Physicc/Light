\doxysubsection{Detailed Description}
\subsubsection*{template$<$typename Entity$>$\newline
class entt\+::basic\+\_\+sparse\+\_\+set$<$ Entity $>$}

Basic sparse set implementation. 

Sparse set or packed array or whatever is the name users give it.~\newline
 Two arrays\+: an {\itshape external} one and an {\itshape internal} one; a {\itshape sparse} one and a {\itshape packed} one; one used for direct access through contiguous memory, the other one used to get the data through an extra level of indirection.~\newline
 This is largely used by the registry to offer users the fastest access ever to the components. Views and groups in general are almost entirely designed around sparse sets.

This type of data structure is widely documented in the literature and on the web. This is nothing more than a customized implementation suitable for the purpose of the framework.

\begin{DoxyNote}{Note}
Internal data structures arrange elements to maximize performance. There are no guarantees that entities are returned in the insertion order when iterate a sparse set. Do not make assumption on the order in any case.
\end{DoxyNote}

Sort the first count elements according to the given comparison function. 

The comparison function object must return {\ttfamily true} if the first element is {\itshape less} than the second one, {\ttfamily false} otherwise. The signature of the comparison function should be equivalent to the following\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{bool(\textcolor{keyword}{const} Entity, \textcolor{keyword}{const} Entity);}
\end{DoxyCode}


Moreover, the comparison function object shall induce a {\itshape strict weak ordering} on the values.

The sort function object must offer a member function template {\ttfamily operator()} that accepts three arguments\+:


\begin{DoxyItemize}
\item An iterator to the first element of the range to sort.
\item An iterator past the last element of the range to sort.
\item A comparison function to use to compare the elements.
\end{DoxyItemize}
