\doxysubsection{Detailed Description}
\subsubsection*{template$<$typename Derived$>$\newline
class entt\+::emitter$<$ Derived $>$}

General purpose event emitter. 

The emitter class template follows the C\+R\+TP idiom. To create a custom emitter type, derived classes must inherit directly from the base class as\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{keyword}{struct }my\_emitter: \mbox{\hyperlink{classentt_1_1emitter_af36907a2dc7cc0a6607533c67e072f92}{emitter}}<my\_emitter> \{}
\DoxyCodeLine{    \textcolor{comment}{// ...}}
\DoxyCodeLine{\}}
\end{DoxyCode}


Pools for the type of events are created internally on the fly. It\textquotesingle{}s not required to specify in advance the full list of accepted types.~\newline
 Moreover, whenever an event is published, an emitter provides the listeners with a reference to itself along with a reference to the event. Therefore listeners have an handy way to work with it without incurring in the need of capturing a reference to the emitter.

Registers a long-\/lived listener with the event emitter. 

This method can be used to register a listener designed to be invoked more than once for the given event type.~\newline
 The connection returned by the method can be freely discarded. It\textquotesingle{}s meant to be used later to disconnect the listener if required.

The listener is as a callable object that can be moved and the type of which is {\itshape compatible} with {\ttfamily void(\+Event \&, Derived \&)}.

\begin{DoxyNote}{Note}
Whenever an event is emitted, the emitter provides the listener with a reference to the derived class. Listeners don\textquotesingle{}t have to capture those instances for later uses.
\end{DoxyNote}
