\subsubsection*{template$<$typename Entity, typename... Exclude, typename... Get$>$\newline
class entt\+::basic\+\_\+group$<$ Entity, exclude\+\_\+t$<$ Exclude... $>$, get\+\_\+t$<$ Get... $>$ $>$}

Non-\/owning group. 

A non-\/owning group returns all entities and only the entities that have at least the given components. Moreover, it\textquotesingle{}s guaranteed that the entity list is tightly packed in memory for fast iterations.

{\bfseries{Important}} 

Iterators aren\textquotesingle{}t invalidated if\+:


\begin{DoxyItemize}
\item New instances of the given components are created and assigned to entities.
\item The entity currently pointed is modified (as an example, if one of the given components is removed from the entity to which the iterator points).
\item The entity currently pointed is destroyed.
\end{DoxyItemize}

In all other cases, modifying the pools iterated by the group in any way invalidates all the iterators and using them results in undefined behavior.

\begin{DoxyNote}{Note}
Groups share references to the underlying data structures of the registry that generated them. Therefore any change to the entities and to the components made by means of the registry are immediately reflected by all the groups.~\newline
 Moreover, sorting a non-\/owning group affects all the instances of the same group (it means that users don\textquotesingle{}t have to call {\ttfamily sort} on each instance to sort all of them because they {\itshape share} entities and components).
\end{DoxyNote}
\begin{DoxyWarning}{Warning}
Lifetime of a group must not overcome that of the registry that generated it. In any other case, attempting to use a group results in undefined behavior.
\end{DoxyWarning}

Sort the shared pool of entities according to the given component. 

Non-\/owning groups of the same type share with the registry a pool of entities with its own order that doesn\textquotesingle{}t depend on the order of any pool of components. Users can order the underlying data structure so that it respects the order of the pool of the given component.

\begin{DoxyNote}{Note}
The shared pool of entities and thus its order is affected by the changes to each and every pool that it tracks. Therefore changes to those pools can quickly ruin the order imposed to the pool of entities shared between the non-\/owning groups.
\end{DoxyNote}

Sort a group according to the given comparison function. 

Sort the group so that iterating it with a couple of iterators returns entities and components in the expected order. See {\ttfamily begin} and {\ttfamily end} for more details.

The comparison function object must return {\ttfamily true} if the first element is {\itshape less} than the second one, {\ttfamily false} otherwise. The signature of the comparison function should be equivalent to one of the following\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{bool(std::tuple<Component \&...>, std::tuple<Component \&...>);}
\DoxyCodeLine{bool(\textcolor{keyword}{const} Component \&..., \textcolor{keyword}{const} Component \&...);}
\DoxyCodeLine{bool(\textcolor{keyword}{const} Entity, \textcolor{keyword}{const} Entity);}
\end{DoxyCode}


Where {\ttfamily Component} are such that they are iterated by the group.~\newline
 Moreover, the comparison function object shall induce a {\itshape strict weak ordering} on the values.

The sort function oject must offer a member function template {\ttfamily operator()} that accepts three arguments\+:


\begin{DoxyItemize}
\item An iterator to the first element of the range to sort.
\item An iterator past the last element of the range to sort.
\item A comparison function to use to compare the elements.
\end{DoxyItemize}
