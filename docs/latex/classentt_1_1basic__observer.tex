\doxysubsection{Detailed Description}
\subsubsection*{template$<$typename Entity$>$\newline
class entt\+::basic\+\_\+observer$<$ Entity $>$}

Observer. 

An observer returns all the entities and only the entities that fit the requirements of at least one matcher. Moreover, it\textquotesingle{}s guaranteed that the entity list is tightly packed in memory for fast iterations.~\newline
 In general, observers don\textquotesingle{}t stay true to the order of any set of components.

Observers work mainly with two types of matchers, provided through a collector\+:


\begin{DoxyItemize}
\item Observing matcher\+: an observer will return at least all the living entities for which one or more of the given components have been updated and not yet destroyed.
\item Grouping matcher\+: an observer will return at least all the living entities that would have entered the given group if it existed and that would have not yet left it.
\end{DoxyItemize}

If an entity respects the requirements of multiple matchers, it will be returned once and only once by the observer in any case.

Matchers support also filtering by means of a {\itshape where} clause that accepts both a list of types and an exclusion list.~\newline
 Whenever a matcher finds that an entity matches its requirements, the condition of the filter is verified before to register the entity itself. Moreover, a registered entity isn\textquotesingle{}t returned by the observer if the condition set by the filter is broken in the meantime.

{\bfseries{Important}} 

Iterators aren\textquotesingle{}t invalidated if\+:


\begin{DoxyItemize}
\item New instances of the given components are created and assigned to entities.
\item The entity currently pointed is modified (as an example, if one of the given components is removed from the entity to which the iterator points).
\item The entity currently pointed is destroyed.
\end{DoxyItemize}

In all the other cases, modifying the pools of the given components in any way invalidates all the iterators and using them results in undefined behavior.

\begin{DoxyWarning}{Warning}
Lifetime of an observer doesn\textquotesingle{}t necessarily have to overcome that of the registry to which it is connected. However, the observer must be disconnected from the registry before being destroyed to avoid crashes due to dangling pointers.
\end{DoxyWarning}
