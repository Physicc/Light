Dear Im\+Gui outputs 16-\/bit vertex indices by default. Allegro doesn\textquotesingle{}t support them natively, so we have two solutions\+: convert the indices manually in imgui\+\_\+impl\+\_\+allegro5.\+cpp, or compile dear imgui with 32-\/bit indices. You can either modify \mbox{\hyperlink{imconfig_8h_source}{imconfig.\+h}} that comes with Dear Im\+Gui (easier), or set a C++ preprocessor option I\+M\+G\+U\+I\+\_\+\+U\+S\+E\+R\+\_\+\+C\+O\+N\+F\+IG to find to a filename. We are providing {\ttfamily \mbox{\hyperlink{imconfig__allegro5_8h_source}{imconfig\+\_\+allegro5.\+h}}} that enables 32-\/bit indices. Note that the backend supports {\itshape B\+O\+TH} 16-\/bit and 32-\/bit indices, but 32-\/bit indices will be slightly faster as they won\textquotesingle{}t require a manual conversion.\hypertarget{md_LightFramework_libs_imgui_examples_example_allegro5_README_autotoc_md167}{}\doxysection{How to Build}\label{md_LightFramework_libs_imgui_examples_example_allegro5_README_autotoc_md167}
