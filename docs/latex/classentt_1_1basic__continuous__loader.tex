\subsubsection*{template$<$typename Entity$>$\newline
class entt\+::basic\+\_\+continuous\+\_\+loader$<$ Entity $>$}

Utility class for {\itshape continuous loading}. 

A {\itshape continuous loader} is designed to load data from a source registry to a (possibly) non-\/empty destination. The loader can accommodate in a registry more than one snapshot in a sort of {\itshape continuous loading} that updates the destination one step at a time.~\newline
 Identifiers that entities originally had are not transferred to the target. Instead, the loader maps remote identifiers to local ones while restoring a snapshot.~\newline
 An example of use is the implementation of a client-\/server applications with the requirement of transferring somehow parts of the representation side to side.

Restores components and assigns them to the right entities. 

The template parameter list must be exactly the same used during serialization. In the event that the entity to which the component is assigned doesn\textquotesingle{}t exist yet, the loader will take care to create a local counterpart for it.~\newline
 Members can be either data members of type entity\+\_\+type or containers of entities. In both cases, the loader will visit them and update the entities by replacing each one with its local counterpart.
