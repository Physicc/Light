Please review the caveats, deprecations and removals if your project was written against an earlier version of G\+L\+FW 3.\hypertarget{news_features_33}{}\doxysubsection{New features in version 3.\+3}\label{news_features_33}

For more information see \mbox{\hyperlink{input_guide_gamepad}{Gamepad input}}.\hypertarget{news_moltenvk_33}{}\doxysubsubsection{Support for Vulkan on mac\+O\+S via Molten\+VK}\label{news_moltenvk_33}
G\+L\+FW now supports \href{https://moltengl.com/moltenvk/}{\texttt{ Molten\+VK}}, a Vulkan implementation on top of the Metal A\+PI, and its {\ttfamily V\+K\+\_\+\+M\+V\+K\+\_\+macos\+\_\+surface} window surface creation extension. This takes effect both on creation and when the window is moved between monitors. It is related to but different from \mbox{\hyperlink{window_guide_GLFW_COCOA_RETINA_FRAMEBUFFER_hint}{G\+L\+F\+W\+\_\+\+C\+O\+C\+O\+A\+\_\+\+R\+E\+T\+I\+N\+A\+\_\+\+F\+R\+A\+M\+E\+B\+U\+F\+F\+ER}}.

For more information see \mbox{\hyperlink{intro_guide_error_handling}{Error handling}}.\hypertarget{news_init_hints_33}{}\doxysubsubsection{Support for initialization hints}\label{news_init_hints_33}

For more information see \mbox{\hyperlink{window_guide_window_maximize}{Window maximization}}.\hypertarget{news_workarea_33}{}\doxysubsubsection{Query for the monitor work area}\label{news_workarea_33}
G\+L\+FW now supports querying the work area of a monitor, i.\+e. the area not occupied by task bars or global menu bars, with \mbox{\hyperlink{group__monitor_ga5a3ac0f6e3786df47aabccfb5760b31d}{glfw\+Get\+Monitor\+Workarea}}.
